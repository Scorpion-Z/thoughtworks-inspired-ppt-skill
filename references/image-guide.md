# Image Guide

本文件用于规范 PPT 配图、图例和截图处理。

## 基本规则

1. 图片是 PPT 内嵌素材，不自带页码、页脚、标题和角标。
2. 中文 deck 的信息图使用中文标签，英文 deck 使用英文标签。
3. 图片比例先匹配落位，再生成或裁剪。
4. 优先生成结构图、系统关系图、流程图、信息图。
5. 真实业务截图需要保真时，先做留白、背景、缩放和对齐处理。
6. 需要概念表达时，再进行截图重构或 UI 情景图设计。

## 常用比例

| 场景 | 推荐比例 |
|------|----------|
| 封面主视觉 | 16:9 或 21:9 |
| 左文右图 | 16:10 或 4:3 |
| 信息图 | 16:9 或 16:10 |
| 截图美化 | 16:10 |
| 头像或图标组 | 1:1 |
| 多图网格 | 统一 3:2 或统一 16:10 |

## 信息图写法

生成用于 HTML PPT 的信息图时，说明主题、用途、比例和语言。保持简洁、强结构、黑白灰基础、单一强调色。画面中不要出现页码、页脚、标题栏和多余装饰。

## 能力框架图写法

用于展示治理层、业务层、数据层、平台层、应用层、安全运维层之间的关系。风格保持企业级、矢量化、中文标签、比例 16:9。

## 路线图写法

用于展示三阶段推进路径。每个阶段包含任务、产出和里程碑。风格保持强网格、单一强调色、比例 16:9。

## 截图美化写法

将截图放入干净的演示文稿画布中，保留截图内容，不重画文字和数据。增加统一留白、浅灰背景和细边框，比例 16:10。

## 禁用项

- 不生成官方 logo。
- 不生成看似官方授权的品牌模板。
- 不在图片里写大段正文。
- 不在图片里加页脚、页码和演讲标题。
- 不把复杂表格画成不可读的小字图片。
